# Проверка гипотез в Python и анализ A/B-теста


## 📊 Обзор проекта

Цель: Статистически проверить гипотезу о различии средней активности пользователей из Москвы и Санкт-Петербурга в приложении «Яндекс Книги», а также оценить корректность и результаты A/B-теста нового интерфейса интернет-магазина «BitMotion Kit».

Дата выполнения: 30.04.2026


## 📁 Данные

### Часть 1. Яндекс Книги

Основной датасет (`yandex_knigi_data.csv`, 8 784 записи, 4 поля):
- `city` — город (Москва / Санкт-Петербург)
- `puid` — идентификатор пользователя (переименован в `user_id`)
- `hours` — суммарное время активности в часах (чтение и прослушивание)
- `Unnamed: 0` — служебный индекс (удалён при предобработке)

### Часть 2. A/B-тест BitMotion Kit

Участники тестов (`ab_test_participants.csv`, 14 525 записей, 4 поля):
- `user_id` — идентификатор пользователя
- `group` — группа теста (A / B)
- `ab_test` — название теста
- `device` — устройство регистрации

События (`ab_test_events.zip`, 787 286 записей, 4 поля):
- `user_id` — идентификатор пользователя
- `event_dt` — дата и время события
- `event_name` — тип события (registration, purchase, login, product_page и др.)
- `details` — дополнительные данные (стоимость или текстовая метка)


## 📈 Этапы анализа

### Часть 1. Проверка гипотезы о времени активности

1. Загрузка и предобработка данных:
   - Удаление служебного столбца `Unnamed: 0`
   - Переименование `puid` в `user_id`
   - Удаление дубликатов пользователей (244 дубликата)

2. Статистический анализ:
   - Расчёт описательных статистик по группам (Москва vs Санкт-Петербург)
   - Проверка дубликатов для обеспечения независимости выборок
   - Выбор одностороннего t-теста Уэлча (без предположения о равенстве дисперсий)

3. Формулировка гипотез:
   - Нулевая гипотеза (H₀): Среднее время активности в Санкт-Петербурге ≤ Москвы
   - Альтернативная гипотеза (H₁): Среднее время активности в Санкт-Петербурге > Москвы

### Часть 2. Анализ A/B-теста

1. Загрузка и оценка целостности данных:
   - Проверка пропусков и дубликатов
   - Удаление 36 318 дубликатов строк из таблицы событий

2. Проверка корректности проведения теста:
   - Соответствие техническому заданию (только группы A и B)
   - Равномерность распределения пользователей (хи-квадрат тест)
   - Выявление пересечений с конкурирующим тестом (887 пользователей исключены)

3. Фильтрация и подготовка данных:
   - Выделение участников теста `interface_eu_test`
   - Ограничение горизонта анализа — первые 7 дней после регистрации
   - Расчёт лайфтайма и фильтрация событий по 7 дням

4. Оценка достаточности выборки:
   - Базовый показатель конверсии: 30%
   - Ожидаемый прирост: 3 п.п.
   - Уровень значимости: α=0.05
   - Мощность теста: 80%
   - Расчёт минимально необходимого размера группы (3 762)

5. Расчёт конверсии:
   - Группа A (старый интерфейс): 4 952 пользователя, 1 377 покупателей
   - Группа B (новый интерфейс): 5 011 пользователей, 1 480 покупателей

6. Статистический тест:
   - Односторонний z-тест для разности пропорций
   - Проверка гипотезы о превышении конверсии в группе B


## 🔍 Ключевые выводы

### Часть 1. Яндекс Книги

Количество пользователей: Москва 6 234, СПБ 2 306
Средняя активность (часы): Москва 10.88, СПБ 11.26
Стандартное отклонение: Москва 36.85, СПБ 39.83
Медиана: Москва 0.92, СПБ 0.88

Результат t-теста:
- t-статистика = 0.4028
- p-value = 0.3436
- Уровень значимости α = 0.05

Вывод: p-value (0.3436) превышает критический уровень 0.05. **Нет оснований отвергнуть нулевую гипотезу.** Статистически значимого превышения среднего времени активности в Санкт-Петербурге не обнаружено.

Возможные причины:
1. Близость культурных паттернов — оба мегаполиса имеют схожий ритм жизни и доступность цифровых сервисов
2. Высокая внутригрупповая вариация — большой разброс активности маскирует межгрупповые различия
